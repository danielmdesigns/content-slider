Steps to Use

  1. Associate the content-slider.js with page. This file is in the CMS.
  2. Each slider will need a parent container. This will a need class or id to hook into the contentSlider method.
  3. Add $("PARENT ID OR CLASS").contentSlider(); somewhere below the content or within a $(document).ready();
  4. If you want custom options, pass them in through like this $("PARENT ID OR CLASS").contentSlider({autoTimeout: 3000, pauseOnHover: false}); Defaults are listed below.
  5. The area with sliding content will need to be wrapped in an element with the class of ".display", it doesn't matter what type of element, div, ul, whatevs, just needs the class of ".display" and to be block level. The direct children of ".display" will be the slides. Anything can go into the slides. Picture, paragraphs, lists, dancing kittens, whatever you'd like!
  6. For Next/Prev buttons, you need child elements to the parent with the class of ".next" and ".prev", it can be any kind of element, span, div, b, i, you name it. Could even be an li inside a ul! All they need is the class of ".next" and ".prev"
  7. For Thumbnails/Pagination, you need an element with the class ".thumbs", it's children will be the thumbails/pagination. You can put whatever you want in the thumbnails. Numbers for pagination, thumbnails, words, letters, emoticons, etc. etc.
  8. Check out the markup for the examples above.

var defaults = {
